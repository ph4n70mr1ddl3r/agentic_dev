# Roadmap

_Authored by the CEO hat via `forge`._

| # | Phase | Goal | Exit criteria |
|---|---|---|---|
| 0 | **Phase 1: Foundation - Reference & Platform Spec** | Establish domain knowledge base (D365 Financials & Supply Chain), define platform spec JSON schemas, set up CI/CD and GitHub workflow. | Published D365 Financials and Supply Chain reference digests.; Platform spec JSON schemas (entities, fields, forms, workflows, rules, actions) defined and validated.; Mono repo with CI pipeline (lint, test, schema validation) operational.; Contribution model documented and GitHub issue/branch/PR workflow active. |
| 1 | **Phase 2: Engine Core - Metadata, Data, Auth** | Build and test the Metadata Engine (loading/versioning), Schema Engine (DDL generation/migration), and Auth Engine (multi-company RLS, roles). | Metadata Engine can load and validate metadata packages into meta.* tables.; Schema Engine generates ent_* tables with typed columns and extras JSONB; migration rollback works.; Auth Engine enforces company_id RLS on all transactional tables; role-based access control functional.; All engines covered by automated unit and integration tests. |
| 2 | **Phase 3: Workflow, Expression, and UI Engines** | Implement Workflow Engine (JSON-logic conditions + action vocabulary), Expression Engine (field formulas), and generic Frontend Shell/Studio. | Workflow Engine can process rules and actions (set-field, update-record, send-notification, etc.) with audit trail.; Expression Engine evaluates JSON-logic formulas correctly in form/field contexts.; Frontend Shell renders any entity form/list from metadata; Admin Studio allows basic metadata authoring.; Security audit of action vocabulary passes. |
| 3 | **Phase 4: Financials Module - Metadata Authoring & Validation** | Author and test complete Financials metadata (GL, AP, AR, Bank, Financial Dimensions) using D365 reference. | Financials metadata package loaded and validated by Metadata Engine.; Core financial processes (journal entry, invoice, payment, reconciliation) work end-to-end.; Multi-company ledger isolation confirmed via integration tests.; SoD controls for financial actions in place. |
| 4 | **Phase 5: Supply Chain Module - Metadata Authoring & Validation** | Author and test complete Supply Chain metadata (Procurement, Inventory, Sales, Production, Warehouse) using D365 reference. | Supply Chain metadata package loaded and validated.; Core supply chain processes (PO, SO, transfer, picking, receipt) work end-to-end.; Integration between Financials and Supply Chain (cost posting, inventory valuation) verified.; Performance benchmarks met (queries, reporting). |
| 5 | **Phase 6: v1.0 Release - Integration, Performance, Documentation** | Integration testing, performance tuning, comprehensive documentation, and final release. | End-to-end regression suite passes.; Deployment scripts for production (single-company, Docker compose) tested.; User and admin documentation published.; Fit-gap analysis against D365 v1 features completed and signed off. |
