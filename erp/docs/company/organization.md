# Organization

_Authored by the CEO hat via `forge`._

**Mission:** Build a model-driven cloud ERP for Financials and Supply Chain Management, modeled on Microsoft Dynamics 365, using a platform of engines and metadata modules, with quality assured through gated delivery, contracts-first design, and rigorous review.

## Hats

| Hat | Role | Responsibilities |
|---|---|---|
| **CEO** | CEO | Company plan, roadmap, phase gates; Overall coordination and delegation |
| **Product Owner** | Product Owner | Domain prioritization; Reference material curation; Gate approval |
| **Solution Architect** | Solution Architect | Architectural decisions; Engine design; Platform spec schemas |
| **Data Architect** | Data Architect | Storage schema design; Postgres hybrid model; Migrations and RLS |
| **Security Architect** | Security Architect | Auth, SoD, audit; Security policies; Vulnerability reviews |
| **Tech Lead** | Tech Lead | Expression language (JSON-logic); Action vocabulary; Microservice implementation; Engineering standards |
| **Domain Modeler (Financials)** | Domain Modeler (Financials) | Financials metadata modules; D365 Financials reference digest; Financial domain schemas |
| **Domain Modeler (Supply Chain)** | Domain Modeler (Supply Chain) | Supply Chain metadata modules; D365 Supply Chain reference digest; Supply Chain domain schemas |
| **Frontend Developer** | Frontend Developer | Generic UI shell; Admin Studio; Form/list rendering from metadata |
| **QA Engineer** | QA Engineer | Test strategy; Contracts-first validation; Automated test suites; Schema validation CI |
| **DevOps Engineer** | DevOps Engineer | CI/CD pipeline; Docker setup; GitHub automation; Release management |
| **Technical Writer** | Technical Writer | Documentation; ADR compliance; User guides; API docs |
